**An update on recent Claude Code quality reports**
https://www.anthropic.com/engineering/april-23-postmortem

Anthropic published a detailed postmortem explaining why Claude Code felt degraded for many users over the past month.
Anthropic 发布了一份详细的事后分析报告，解释了过去一个月 Claude Code 对许多用户来说体验下降的原因。

As of April 23, Anthropic is resetting usage limits for all subscribers and implementing tighter controls on future system prompt changes.
自 4 月 23 日起，Anthropic 已为所有订阅用户重置使用限额，并对未来的系统提示变更实施更严格的管控。

**Built-in memory for Claude Managed Agents**
https://claude.com/blog/claude-managed-agents-memory

Memory on Claude Managed Agents is now in public beta, allowing agents to learn from every session and share knowledge across sessions.
Claude Managed Agents 的内置记忆功能现已进入公测阶段，支持 Agent 在每次会话中持续学习并跨会话共享知识。

Memories are stored as files on a filesystem, giving developers full export and API control — including scoped permissions, audit logs, and rollback capabilities.
记忆以文件形式存储在文件系统中，开发者可通过 API 完整控制，包括权限范围设置、审计日志和版本回滚功能。

Early adopters include Netflix, Rakuten (cutting first-pass errors by 97%), and Wisedocs (speeding document verification by 30%).
早期采用者包括 Netflix、Rakuten（将首次错误率降低 97%）和 Wisedocs（文档核验速度提升 30%）。
