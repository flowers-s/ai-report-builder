Garry Tan 在其开源项目被数万 agentic 工程师使用3个月后，抛出了一个架构哲学宣言：*"薄 harness，厚 skills。"* Harness 应是一个轻量指挥者——负责读取文件、路由上下文、调用工具；技能与记忆存储在 Markdown 里，存储在 Git 里。*"如果你的 harness 挂掉，记忆也跟着消失，说明你的 harness 做得太厚了。"* 这是对当前 Agent 框架设计的直接批判，也是一种架构纪律的倡导：复杂性要放在对的地方。他还提到通过 AI 构建语音系统是一次关于延迟的深刻教育。
Garry dropped a philosophy bomb on agentic architecture after 3 months of open-source work used by tens of thousands of engineers daily. His thesis: *"Thin harness, fat skills."* The harness should be a thin conductor — it reads files, routes context, calls tools. Skills and memory live in markdown, in git. *"If your memory dies when your harness dies, you built the harness too thick."* He also praised the learning value of building voice systems with AI — calling it *"a powerful education in latency."*

Peter 公布了下一版 OpenClaw 的两项实验，直击"GPT 太懒"的用户痛点。其一，**严格模式**：通过新的执行契约（`strict-agentic`）要求 GPT-5.x 持续自主推进，不轻易停下来等待指令。其二，**原生 Codex 作为 harness**：插件化架构允许将默认的 Pi harness 替换为 Anthropic SDK 或 Codex，大幅提升可组合性。Harness 插件化是一个重要的架构信号——它与 @garrytan 的"薄 harness"哲学形成呼应，让生态更开放。
Peter unveiled two experiments for the next OpenClaw release addressing "GPT is lazy" complaints. First, **Strict Mode**: a new execution contract (`strict-agentic`) that forces GPT-5.x to keep working autonomously. Second, **Native Codex as harness**: a plugin system where the default Pi harness can be swapped for Anthropic's SDK or Codex. He also noted that harnesses can now be plugins — making the architecture far more composable, even if Claude subscription support isn't there yet.

Aditya 在"真正的 Agent"与"循环调用 LLM"之间划出了一条清晰的分界线：真正的区别在于**智能的长时记忆管理**。如何决定记什么、忘什么、如何压缩与检索跨越长任务周期的上下文——这是 agentic AI 中最难、也最值得关注的未解问题。记忆不是附加功能，而是 Agent 智能的核心基础设施。
Aditya drew a sharp line between a true "agent" and simply running an LLM in a loop: the real differentiator is **intelligent long-horizon memory management**. Getting this right — what to remember, what to forget, how to compress and retrieve context across long task horizons — is the hardest and most interesting unsolved problem in agentic AI.
