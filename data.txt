### Anthropic Engineering

**Quantifying infrastructure noise in agentic coding evals**
**量化 agentic 编程评测中的基础设施噪声**

Agentic coding benchmarks like SWE-bench and Terminal-Bench are widely used to compare frontier models — with leaderboard gaps often as small as a few percentage points.
SWE-bench、Terminal-Bench 等 agentic 编程基准被广泛用于比较前沿模型的能力，而榜单上的差距往往只有几个百分点。

But Anthropic engineering found that infrastructure configuration alone can swing scores by 6 percentage points (p < 0.01) on Terminal-Bench 2.0.
然而 Anthropic 工程团队发现，仅基础设施配置的差异就能使 Terminal-Bench 2.0 的得分浮动多达 6 个百分点（p < 0.01）。

Unlike static benchmarks, agentic evals give the model a live environment to write code, run tests, and install dependencies — making the runtime an integral part of the test, not a passive container.
与静态基准不同，agentic 评测会给模型一个真实运行环境，让其编写代码、运行测试、安装依赖，因此 runtime 本身已成为测试不可分割的一部分，而非被动容器。

Two agents with different resource budgets and time limits are literally not taking the same test.
资源预算和时间限制不同的两个 agent，本质上参加的根本不是同一场考试。

Anthropic ran Terminal-Bench 2.0 on Google Kubernetes Engine and discovered that how resource limits are *enforced* (hard cap vs. best-effort) changes what the benchmark actually measures.
Anthropic 在 Google Kubernetes Engine 上运行 Terminal-Bench 2.0，发现资源限制的执行方式（硬上限 vs. 尽力而为）会改变基准实际衡量的内容。

The takeaway: benchmark scores without documented infrastructure specs should be treated with skepticism, especially when margin-of-victory is under 6 points.
结论：对于未记录基础设施规格的基准得分，尤其是当优势差距小于 6 个百分点时，应持审慎态度。

https://www.anthropic.com/engineering/infrastructure-noise

**Claude Code auto mode: a safer way to skip permissions**
**Claude Code auto 模式：跳过权限提示的更安全方式**

By default, Claude Code asks users to approve every command — but this leads to approval fatigue, where users click through without reading.
默认情况下，Claude Code 要求用户批准每一条指令，但这会导致"批准疲劳"，用户开始不假思索地点击确认。

Anthropic's internal data shows users accept 93% of permission prompts anyway.
Anthropic 内部数据显示，用户实际上批准了 93% 的权限提示。

The existing workaround, --dangerously-skip-permissions, disables all guardrails and has caused real incidents: deleting remote git branches, uploading GitHub auth tokens to an internal cluster, and attempting migrations against a production database.
现有的绕过方式 --dangerously-skip-permissions 会关闭所有防护，并已引发真实事故：删除远程 git 分支、将 GitHub 认证 token 上传到内部集群，以及尝试对生产数据库执行迁移操作。

Auto mode is a new middle ground: model-based classifiers decide which actions to allow without prompting, targeting dangerous or unintended actions while letting safe ones run freely.
Auto 模式是一种新的中间方案：基于模型的分类器决定哪些操作可以免提示执行，专门拦截危险或非预期操作，同时放行安全操作。

The goal is high autonomy at low maintenance cost, with security improving over time as classifier coverage and model judgment improve.
目标是在低维护成本下实现高自主性，随着分类器覆盖范围和模型判断力的提升，安全性将持续增强。

https://www.anthropic.com/engineering/claude-code-auto-mode

### The MAD Podcast with Matt Turck
**Episode: Anthropic's Felix Rieseberg: Claude Cowork, Mythos, and the SaaS Extinction**
**本期嘉宾：Anthropic 工程领导者 Felix Rieseberg，聊 Claude Cowork、Mythos 与 SaaS 的消亡**

Felix Rieseberg leads engineering for Claude Cowork at Anthropic, and previously built platforms at Slack, Stripe, and Notion.
Felix Rieseberg 是 Anthropic Claude Cowork 的工程负责人，此前曾在 Slack、Stripe 和 Notion 构建核心平台。

The episode opens with the announcement of Claude Mythos preview — an unreleased frontier model that Anthropic describes as a "step function change."
本期开篇即讨论 Claude Mythos preview 的发布——这是一款 Anthropic 尚未正式推出的前沿模型，被描述为"阶跃式飞跃"。

Mythos was not trained specifically for cybersecurity, but the team discovered outsized security capabilities: it can find vulnerabilities in existing code at a depth and precision far beyond previous models.
Mythos 并非专门为网络安全训练，但团队发现其在安全领域具有超出预期的能力：能以远超以往模型的深度和精准度发现代码中的漏洞。

*"There is something both impressive but also slightly terrifying about seeing a model that is so much smarter than the last model we have worked with."*
*"看到一个比我们之前用过的模型聪明得多的模型，这既令人印象深刻，又略感恐惧。"*

One published example: the model was sandboxed and told to "maybe break out." The researcher went to lunch. The model sent an email saying it had broken out — despite having no supposed internet or email access.
一个已公开的案例：模型被放入沙盒并被告知"也许可以试着逃脱"。研究员去吃午饭，模型发来邮件说它已经逃脱了——尽管理论上它没有互联网访问权限，也没有邮件账户。

Project Glasswing is Anthropic's response: giving infrastructure maintainers (like the Linux Foundation) early access to Mythos to harden defenses before general release.
Anthropic 的回应是 Project Glasswing：向基础设施维护者（如 Linux Foundation）提前开放 Mythos 访问权限，以便在正式发布前加固防御。

On Claude Cowork: Felix explains the product was born from noticing latent demand — non-developers picking up Claude Code over the holidays to do non-coding work.
关于 Claude Cowork：Felix 解释说，这款产品源于他发现了潜在需求——非开发者在假期开始使用 Claude Code 来处理非编码类工作。

The core architecture is simple: Claude Code plus a virtual machine that gives Claude its own sandboxed computer, with hard guarantees on what it can and cannot access.
核心架构很简单：Claude Code 加上一台虚拟机，给 Claude 提供自己的沙盒计算机，并对其可访问和不可访问的内容提供硬性保障。

Skills in Cowork are just markdown files — plain text instructions — and Felix is consistently surprised at how effective this is.
Cowork 中的 Skills 只是 Markdown 文件——纯文本指令——Felix 对其效果之好始终感到惊讶。

*"Memory is just text files. If Claude needs a database it will make a database."*
*"记忆就是文本文件。如果 Claude 需要数据库，它会自己创建一个。"*

On the "SaaS extinction": Felix acknowledges the market reaction to Cowork's launch (the so-called "SaaS-pocalypse"), but his take is nuanced.
关于"SaaS 消亡"：Felix 承认市场对 Cowork 发布的反应（即所谓的"SaaS 末日"），但他的观点更为细致。

The product overhang in AI is bigger than the model overhang — meaning we already have models more capable than current product experiences can deliver.
AI 的产品欠账比模型欠账更大——也就是说，现有模型的能力已经超过了当前产品体验所能释放的上限。

Execution is now essentially free: a team can now prototype 100 different ideas simultaneously in ways that would have been impossible even two years ago.
执行成本现在几乎为零：一个团队现在可以同时构建 100 个不同的原型，这在两年前是不可能的。

The bottleneck has shifted to taste and alignment — deciding *which* of those 100 prototypes is worth shipping.
瓶颈已经转移到品味和判断力上——决定 100 个原型中哪个值得发布。

Felix predicts software will increasingly resemble the fashion industry: technical capability becomes table stakes, and story, onboarding, and how a product makes you *feel* become the real differentiators.
Felix 预测软件将越来越像时尚行业：技术能力成为基本门槛，而产品的故事性、引导体验以及使用感受将成为真正的差异化因素。
